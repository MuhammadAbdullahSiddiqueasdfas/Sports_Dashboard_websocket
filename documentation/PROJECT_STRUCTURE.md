# Project Structure Overview

Here is the high-level organization of the `Sports_dashboard` project.

```text
Sports_dashboard/
├── drizzle/                # SQL migration files generated by Drizzle Kit
├── src/                    # All source code lives here
│   ├── data/               # Static data files (e.g., data.json for seeding)
│   ├── db/                 # Database connection and schema definitions
│   ├── routes/             # Express route handlers (API endpoints)
│   ├── seed/               # Database seeding logic
│   ├── utils/              # Helper/Utility functions
│   ├── validation/         # Zod schemas for validating incoming data
│   ├── ws/                 # WebSocket server implementation
│   ├── arcjet.js           # Security configuration
│   └── index.js            # Main entry point (starts the server)
├── .env                    # Secret environment variables (DB URL, API keys)
├── .env.example            # Template for the .env file
├── drizzle.config.js       # Configuration for Drizzle Kit
├── package.json            # Project dependencies and scripts
└── test_api.js             # Script to test the API endpoints automatically
```

## Folder Details:

### `src/db/`
Contains `db.js` which sets up the connection pool to PostgreSQL, and `schema.js` which defines what our tables (Matches, Commentary) look like.

### `src/routes/`
Contains the logic for different parts of our API. For example, `matches.js` handles everything related to fetching or creating sports matches.

### `src/ws/`
Handles the real-time portion of the app. It manages "subscriptions" where clients can listen for updates on specific matches.

### `src/validation/`
Defines "rules" for our data. If a user tries to create a match without a team name, Zod will catch it here before it even hits the database.
